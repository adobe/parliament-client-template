                                      SDS API Documentation


                                      This documentation describes SDS APIs, that can be broadly categorized into : Index, Ops, ResolveBy. For any questions please contact acp-cs-sds-engineering@adobe.com

                                      Version: "Version Date: 04/22/2020"#

                                      /assets#

                                      GET#

                                      Summary:#

                                      Discoverable Assets Document request

                                      Description:#

                                      Returns Discoverable Assets Document or 'responsetoolarge' error if the count of discoverable assets exceeds max page size

                                      /assets/:page#

                                      GET#

                                      Summary:#

                                      Discoverable Assets Document paginated request

                                      Description:#

                                      Returns user discoverable assets page by page _embedded entries in children always include api:metadata/repository resource. Based on the embed query parameter value, a user agent can request embedding of api:ac/effective and api:rel/repository resources

                                      Parameters#
                                      NameLocated inDescriptionRequiredSchema
                                      startqueryToken value for Page StartNostring
                                      limitqueryNumber of entries requestedNostring
                                      AuthorizationheaderUser Authorization token: (e.g. Bearer XXXXXXXXXXXX...)Yesstring
                                      x-api-keyheaderAPI Gateway API key, which must be white listed ahead of time to be allowedYesstring
                                      embedqueryAllowed link relations are - api:rel/repository, api:ac/effective. An example = http://ns.adobe.com/adobecloud/rel/repositoryNostring
                                      Responses#
                                      CodeDescriptionSchema
                                      200The _links map includes entries for api:page and api:primaryDiscoverableAssets
                                      400if any invalid input arguments in the requestProblem
                                      500if internal or upstream service errorProblem

                                      /content/ops#

                                      POST#

                                      Summary:#

                                      API for Discard/Restore/Package/Copy/Move/Delete operations

                                      Description:#

                                      When success, returns 200 for Discard/Restore operations, 202 for Copy/Package operations 202/204 for delete operations. For 202, client needs to wait on monitor URL returned via Header "Location" until monitor URL returns non 202 status

                                      Parameters#
                                      NameLocated inDescriptionRequiredSchema
                                      AuthorizationheaderUser Authorization token: (e.g. Bearer XXXXXXXXXXXX...)Yesstring
                                      X-Api-KeyheaderAPI Gateway API key, which must be white listed ahead of time to be allowedYesstring
                                      If-MatchheaderPerform operation on directory only if matching this etag value for single opsNostring
                                      Ops requestbodyOps request payload in Json schema, either an array of requests or a single request. Note: Model/Schema shown here for single request only, but API accepts an array of requestsYesOpsRequestPayload
                                      Responses#
                                      CodeDescriptionSchema
                                      200OK - For Discard and Restore operations. 202/204 for Other operationsOpsResponse
                                      202Accepted - For Copy, Delete, Package operationsOpsResponse
                                      204No Content for Delete operationsOpsResponse
                                      307if request is temporarily redirectedBaseResponse
                                      400if the repo id provided is empty or invalid / source or target fields not validBaseResponse
                                      401if user token is not validBaseResponse
                                      403if user does not have permission to perform the operation or directory state is discardedBaseResponse
                                      404if directory does not exist for given source pathBaseResponse
                                      409if directory already exist for given target pathBaseResponse
                                      412if etag is not validBaseResponse

                                      /content/resolve#

                                      GET#

                                      Summary:#

                                      Handle ResolveBy requests either by assetId or path

                                      Description:#

                                      Returns the resolved assets including URL templates

                                      Parameters#
                                      NameLocated inDescriptionRequiredSchema
                                      AuthorizationheaderUser Authorization token: (e.g. Bearer XXXXXXXXXXXX...)Yesstring
                                      X-Api-KeyheaderAPI Gateway API key, which must be white listed ahead of time to be allowedYesstring
                                      If-None-MatchheaderA list of ETags which will be compared to the one for this resource. If any match, a 304 will be returned without data and if none match or not provided, it will return the resourceNostring
                                      idqueryDirectory Id to resolve by.Nostring
                                      repositoryIdqueryRepositoryId of the resolve request, and mandatory for ResolveBy Path onlyNostring
                                      pathqueryDirectory Path to resolve by. Only one of the arguments - assetId or path can be specifiedNostring
                                      Responses#
                                      CodeDescriptionSchema
                                      200okDirectoryResponse
                                      304if if-none-match provided and matches etag on resourceBaseResponse
                                      404if the asset does not existBaseResponse

                                      /index#

                                      GET#

                                      Summary:#

                                      Index Document request

                                      Description:#

                                      Returns user's assigned directory, user's repository and regions where discoverable assets exist Also includes links at top level for api:resolve/id, api:resolve/path and api:ops requests

                                      Parameters#
                                      NameLocated inDescriptionRequiredSchema
                                      AuthorizationheaderUser Authorization token: (e.g. Bearer XXXXXXXXXXXX...)Yesstring
                                      x-api-keyheaderAPI Gateway API key, which must be white listed ahead of time to be allowedYesstring
                                      If-None-MatchheaderA list of ETags which will be compared to the one for this resource. If any match, a 304 will be returned without data and if none match or not provided, it will return the resourceNostring
                                      Responses#
                                      CodeDescriptionSchema
                                      200The _links map includes entries for api:resolve/id, api:resolve/path, api:opsIndexDocument
                                      304if if-none-match provided and matches etag on resourceProblem

                                      OpsRequestPayload#

                                      NameTypeDescriptionRequired
                                      opstringSpecifies Operation. Allowed values(case insensitive) = Copy, Move, Discard, Restore, Delete, PackageYes
                                      idstringuser agent-assigned identifier for this operationNo
                                      sourceOpsRefThis is the schema for operations source or target. Only one of the properties path / assetId / href can be specified . Please Note: 1. href as an asset reference for a source or a target is sufficient in itself 2. repo:assetId parameter can be specified as an asset reference for a source or a target. 3.When repo:path parameter is specified to refer to an asset, repo:repositoryId must be specifiedNo
                                      targetOpsRefThis is the schema for operations source or target. Only one of the properties path / assetId / href can be specified . Please Note: 1. href as an asset reference for a source or a target is sufficient in itself 2. repo:assetId parameter can be specified as an asset reference for a source or a target. 3.When repo:path parameter is specified to refer to an asset, repo:repositoryId must be specifiedYes
                                      valueOpsValue@Deprecated: This is the schema for a ops valueNo
                                      maxpackagesizestringSpecifies max package size, applicable for Package operationNo
                                      intermediatesbooleantrue if target intermediate directory hierarchy to be created, applicable for Copy and Move OperationsNo
                                      recursivebooleantrue if operation to be done recursively, applicable for delete onlyNo
                                      namestringSpecifies new name for Restore, applicable only for restore onlyNo
